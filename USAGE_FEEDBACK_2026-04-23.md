# Замечания после использования VVAM (2026-04-23)

Контекст: транскрипция 7 коротких голосов/видео (7–41 сек) — клиентские баг-репорты по shop (ru).
Использовалась модель `gemini-2.5-flash-native-audio-latest` через Live API, `transcriber=gemini`.

---

## 1. Протухший ключ не отлавливается заранее

- Ключ из `tasks/A4_test_and_fix.md` выдал `APIError 1007: API key expired` **на каждый** из 7 файлов подряд, прежде чем я вручную подставил запасной из `test_transcription.py`.
- Нужна проверка ключа одним дешёвым запросом **до** начала обработки (ping `models.list` или 1-секундный тест).
- Либо поддержка списка ключей в конфиге с автоматическим fallback при 1007/429.

## 2. Транзиентный `abnormal closure [internal]` (1006) без ретрая

- Файл `413.wav` на первом прогоне упал с `1006 abnormal closure` в середине Live-сессии. Второй ручной запуск — успех с тем же ключом, тем же файлом.
- Это типичная сетевая/серверная флейковость Live API. Нужен встроенный retry (1–3 попытки с экспоненциальным бэкоффом) как минимум для 1006.

## 3. Нет батча с сохранением в файлы

- `voice <dir> --all` работает, но сливает всё в один выхлоп. Чтобы получить **по файлу на источник**, пришлось делать внешний for-цикл с `-o vvam_output/.../NNN.md`.
- Удобно было бы: `voice <dir> --all --output-dir path/ --name-template "{stem}.md"`.

## 4. Разные плейсхолдеры для «тишины»

- `414.wav` (скринкаст без речи) → вернул **`. . . .`** (точки с пробелами).
- `422.wav` (тоже скринкаст) → вернул **`[noise]`**.
- Один и тот же тип контента — два разных маркера. Нужно привести к одному (`[no_speech]` или пустая строка), иначе downstream-обработчикам приходится матчить оба.

## 5. Ошибки «пишу файл, но файл пустой»

- При `APIError` в режиме `-o` файл **не создаётся** и нет `.err`-лога рядом. Приходится парсить stdout/trace чтобы понять, какие из N файлов реально готовы.
- Предложение: при любом фейле писать `NNN.md.err` с кодом ошибки — тогда видно, что обработано, что нет.

## 6. Качество RU-распознавания — середина

На коротких голосовухах без жаргона — нормально. Проблемы:

- `417`: "Ну **истина** бесконечная загрузка" — вероятно было "истинно" или "у меня тут". Грамматика ломается.
- `418` (сложный, 41s): пропуски, "Ээ" филлеры, обрывки — клиент реально рваным темпом, но местами видно что модель склеила две фразы в одну.
- Специфичный жаргон (DevTools, disabled, дозагрузка, аксессуары) — угадывает, но не всегда: например "загрузить ещё" прошло, а отрицательная конструкция "просто ошибка теперь **не** выдаёт" прозвучала двусмысленно.

## 7. Нет `--prompt` / glossary для voice

- Для `video` в CLI есть `--prompt`. Для `voice` — нет.
- Если бы я мог подать контекст ("магазин одежды, жаргон: DevTools, disabled, дозагрузка, корзина, аксессуары, VPN"), пункт 6 стал бы заметно лучше.

## 8. Нет `--lang` для voice

- Автоопределение на русский работает, но коротким (≤7 сек) голосовухам иногда не хватает материала. Явный `--lang ru` ускорил бы и повысил стабильность.
- В конфиге `whisper_language: "ru"` есть, а для gemini — нет.

## 9. Таймкоды крупными блоками

- В markdown-формате сегменты идут по 30 сек; внутри сегмента — один абзац без разметки по предложениям. В 41-секундном сообщении получилось всего 2 строки таблицы.
- Для сверки с видео удобнее было бы дробить по паузам/предложениям (VAD уже включён — можно эмитить таймкоды на каждом `activity_end`).

## 10. Live API — избыточен для коротких файлов

- Все мои файлы — ≤41 секунды. Live API с WebSocket и 30-сек сегментацией здесь overkill: больше поводов для 1006, медленнее из-за `asyncio.sleep(0.25)` на чанк (2× реалтайм).
- Для файлов ≤60 сек стоило бы иметь **fast path** через обычный `gemini-2.5-flash` inline-аудио (один HTTP). Работает быстрее и надёжнее.

## 11. Параллелизма в batch нет

- 7 файлов × ~150% realtime ≈ ~3–4 минуты. Gemini квоту позволяет гонять несколько сессий параллельно (у меня две сессии не коллизили). `asyncio.gather` по файлам в `voice --all` сэкономил бы 2–3× времени.

## 12. Логи засорены трейсбеками

- При 1007/1006 в stdout сыпется полный Python traceback, хотя из CLI нужно было бы одно строчное сообщение вида `ERROR 1007: API key expired — set VVAM_GEMINI_API_KEY`.
- Трейс полезен при `--verbose`, по умолчанию — сворачивать.

## TL;DR — что я бы сделал первым

1. **Pre-flight ping ключа** + fallback на список ключей (покрывает пункт 1).
2. **Retry x3 с бэкоффом** для 1006/временных сетевых (покрывает пункт 2).
3. **Fast path** через inline-аудио на `gemini-2.5-flash` для файлов ≤60 сек (покрывает пункты 2, 10, 11).
4. **Унификация плейсхолдера тишины** + `.err`-файлы при фейле (покрывает пункты 4, 5).
5. **Флаги `--prompt` и `--lang` для `voice`** (покрывает пункты 7, 8).
